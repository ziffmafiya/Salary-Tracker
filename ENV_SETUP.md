# Salary Tracker - Environment Variables Setup

## For Vercel Deployment

On Vercel the app reads credentials at runtime via `/api/config` (rewritten from `/config.js`).

1. Project → **Settings** → **Environment Variables**
2. Add for **Production**, **Preview**, and **Development**:

| Variable Name | Value |
|--------------|-------|
| `SUPABASE_URL` | `https://xxxx.supabase.co` |
| `SUPABASE_ANON_KEY` | your anon/public key |

3. Redeploy after changing variables.

`VITE_SUPABASE_*` names also work if you already use them.

## For Local Development

1. Copy `config.example.js` → `config.js`
2. Fill in your URL and anon key
3. Open `index.html` via a local server (or open the file)

`config.js` is git-ignored. On Vercel it is generated by the API, not from the repo.

**Never** commit real keys. **Never** put the `service_role` key in the client.
